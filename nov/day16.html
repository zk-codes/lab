Ten

This page uses exactly ten HTML element types. Not including the structural tags (html, head, body, meta, title, link), only ten tags build everything you see: article, h1, p, em, strong, blockquote, hr, a, span, aside.

The constraint forces ruthless prioritization. No lists, no sections, no divs. Every element must earn its place. What can you say with only ten words? What can you build with only ten tools?

Constraint is the mother of invention.

Most pages use dozens of element types. Headers nest within navs within divs within sections. But HTML was designed to be semantic. Each tag has meaning. Ten tags, used well, can say everything that needs saying.

Count them: article wraps, h1 announces, p carries prose, em emphasizes, strong strengthens, blockquote quotes, hr divides, a links, span highlights, aside notes. Ten is enough.
