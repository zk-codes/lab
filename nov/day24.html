Natural Order

stacking without z-index

The z-index property controls stacking order. But the browser already has rules for this. Later elements in the DOM stack above earlier ones. Position establishes stacking contexts. The order of appearance is the order of layers.

These three boxes have no z-index. They stack naturally: the element that appears later in the HTML sits on top. Layer 1 is in the DOM first, so it's at the bottom. Layer 3 appears last, so it's on top.

The same principle works for any overlapping layout. Put what should be behind first in the HTML. Put what should be in front last. The document order becomes the visual order.

Z-index is powerful, but it introduces complexity: stacking contexts, integer wars, "just add 9999." The natural order is simpler. Structure your HTML thoughtfully, and the layers fall into place.

This constraint forces us to think about document structure as visual structure. The order of our markup matters not just for accessibility and semantics, but for visual composition too.

No z-index appears anywhere in this stylesheet. The layers work because the DOM works. The browser's default behavior is often enough.
