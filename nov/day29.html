Classful

styling without IDs

This page contains no ID selectors. Not in the HTML, not in the CSS. Every style is applied through classes or element selectors.

Why Avoid IDs?

Specificity

IDs have high specificity, making them hard to override without !important.

Reusability

IDs must be unique. Classes can be reused across multiple elements.

Consistency

Using only classes creates a more predictable, maintainable codebase.

Modern CSS methodologies like BEM and utility-first frameworks deliberately avoid ID selectors. The constraint is already mainstream in professional development.

/* Instead of */
#header { ... }
#main-content { ... }

/* We use */
.page-header { ... }
.content-block { ... }
IDs were designed for JavaScript hooks and anchor links, not for styling. Keeping them separate keeps your CSS cleaner and more modular.

The constraint reveals nothing is lost. Every layout, every style, every effect on this page works without a single ID. Classes are enough.
