The Classless Stylesheet

Styling without a single class selector

Before the proliferation of utility classes and component frameworks, CSS had a simpler vocabulary. Element selectors. Type selectors. The occasional ID. And it worked.

This page contains zero class selectors. Every style you see comes from targeting elements directly, using attribute selectors, or leveraging the cascade. It's a return to fundamentals.

The Tools We Have

Without classes, we rely on semantic structure. The hierarchy of headings. The relationship between parent and child. The order of siblings. CSS gives us powerful selectors:

article > p:first-of-type::first-line { }
a[href^="http"]::after { }
input[type="text"]:focus { }
section > blockquote + p { }
The absence of classes forces us to think about HTML semantically. Every element earns its style through what it is, not what we label it.
